La relation Accès modèlise l'accès des concepts comportementaux aux objets métier ou aux objets données.

La relation Accès indique qu'un processus, une fonction, une interaction, un service ou un événement "fait quelque chose" avec un élément de structure passif; par exemple, crée un nouvel objet, lit des données d'un objet, écrit ou modifie les données de l'objet données ou supprime l'objet. La relation peut également être utilisée pour indiquer que l'objet est simplement associé avec le comportement.

La flèche indique le flux d'information.

Categorie:

Structurel.

Exemples:

Créer un nouvel objet, lire les données d'un objet, écrire ou modifier les données d'un objet, supprimer un objet.
