Dossiers et organisation

Un modèle dans Archi est organisé sous forme d'une structure de dossiers représentant les couches d'ArchiMate et les relations entre les éléments. Un nouveau modèle comporte les dossiers de premier niveau suivant:

Métier Contient les éléments de la couche "Métier" et tous les sous-dossiers créés par un utilisateur
Application Contient les éléments de la couche "Application" et tous les sous-dossiers créés par un utilisateur
Technologie Contient les éléments de la couche "Technologie" et tous les sous-dossiers créés par un utilisateur
Motivation Contient les éléments de la catégorie "Motivation" et tous les sous-dossiers créés par un utilisateur
Implémentation & Migration Contient les éléments de la catégorie "Implémentation & migration" et tous les sous-dossiers créés par un utilisateur
Autre Contient les éléments de type emplacement, regroupement et jonction et tous les sous-dossiers créés par un utilisateur
Relations Contient les relations entre les éléments telles qu'elles ont été créées dans les vues (diagrammes) et tous les sous-dossiers créés par un utilisateur
Vues Contient les liens vers les vues (diagrammes)

Les éléments peuvent être créés et supprimés directement dans l'arborescence des modèles (voir Ajouter des éléments directement dans l'arborescence des modèles) ou sont automatiquement ajoutés dans le type approprié de répoertoire quand des objets sont glissés-déposés sur le canevas d'une vue (voir Ajouter de nouveaux éléments à la vue depuis la palette). Tous les éléments dans un dossier sont automatiquement triés alphabétiquement.


Sous-dossiers créés par un utilisateur

Les sous-dossiers des utilisateurs peuvent être créés sous les dossiers de premier niveau. Ceci vous permet d'organiser les éléments de la manière que vous voulez. Pour ajouter un nouveau sous-dossier à l'arborescence des modèles, sélectionnez un dossier de premier niveau (ou un sous-dossier créé par un utilisateur) et faites un clic-droit. Un "nouveau" élément de menu vous permet d'ajouter un nouveau sous-dossier à l'arborescence.

Notez qu'un sous-dossier ne peut contenir que des éléments du même type que le dossier parent de premier niveau. Par exemple, seuls les éléments de type "Métier" peuvent être créés dans le dossier "Métier" ou l'un de ses sous-dossiers.

Vous pouvez également glisser-déposer des éléments et des sous-dossiers dans la même branche de dossiers, mais pas entre des dossiers de types différents.

Pour renommer un sous-dossier dans l'arborescence des modèles, choisissez "Renommer" dans le menu principal "Édition" ou via le menu contextuel qui apparait en faisant un clic droit.
